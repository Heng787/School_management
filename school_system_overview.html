School Management System

A simple guide to how the system works — no technical knowledge needed!

👥 Part 1 — Who Uses the System?

These are the real people who log in every day and use the app to do their jobs.

👨‍💼

School Admin

The boss of the system. Can see and manage everything — students, staff, reports, and settings.

👩‍🏫

Teacher

Views their classes, marks student attendance, enters grades, and sends messages to the admin.

👩‍💻

Office Worker

Handles student information, registration, and helps manage the school schedule.

⚙️ Part 2 — How Does It Work?

Think of the system as 4 layers — like floors in a building, each with its own job.

🖥️

The App (What you see)

This is the website you open in your browser. It has 8 different pages — Dashboard, Students, Staff, Classes, Reports, Schedule, Settings, and Messages.

Like: The front desk of a school that everyone walks into.

🧠

The Brain (DataContext)

A hidden layer that holds all the information in memory while you're using the app. When you change something, it instantly updates everywhere on screen.

Like: The school office that knows where every student file is.

📡

The Messenger (Services)

A set of helpers that talk to the database and save your changes. When you add a student, the messenger carries that info to the database for permanent storage.

Like: A courier that carries letters between the office and the filing room.

⚡

The Flash Engine (localStorage)

The primary storage engine running natively on your computer. It stores all students, grades, and settings directly in your browser for zero-latency loading and instant data saving.

Like: A filing cabinet sitting right on your desk for immediate access.

🔒

Local-First Mode

The system is currently running entirely offline. Cloud synchronization has been safely decoupled to prevent slow load times. Your data will never lag or wait for the internet.

Like: A fully self-contained management suite.

📂 Part 3 — What Data Is Stored?

Think of the database as a filing room with different folders — one for each topic.

👨‍🎓 Students

Student Records

Everything about each student.

  • Name, gender, date of birth
  • Phone number
  • Enrollment date
  • Status (Active / Suspended)
👩‍🏫 Staff

Staff Records

Details of every employee.

  • Name & role (Teacher, Guard…)
  • Subject they teach
  • Contact number
  • Date they were hired
  • Login password
🏫 Classes

Class Listings

Details of each class/classroom.

  • Class name & level
  • Which teacher runs it
  • Schedule / timetable
📋 Enrollments

Who's in Which Class

Links students to their classes.

  • Student name
  • Class name
  • Academic year
📝 Grades

Student Grades

Scores for each subject.

  • Student name
  • Subject
  • Score (0–10)
  • Term (Semester 1, 2…)
📅 Attendance

Daily Attendance

Who came to school each day.

  • Student name
  • Date
  • Status: Present / Absent / Late
📆 Events

School Calendar

Important dates for the school.

  • Event title
  • Date
  • Type: Holiday, Exam, Meeting
🏖️ Staff Leave

Leave Requests

When staff take time off.

  • Staff name
  • Type of leave
  • Start & end date
  • Reason given
🚪 Daily Logs

Gate Entry / Exit Log

Recorded by the Guard.

  • Who entered or exited
  • Purpose of visit
  • Time & date
🚨 Incidents

Incident Reports

Security problems reported.

  • Title of incident
  • Description
  • Severity: Low / Medium / High
🧹 Rooms

Room Status

Cleaning updates by Cleaner.

  • Room name
  • Status: Cleaned / Needs Attention
  • Who updated it
💬 Messages

Internal Messages

Staff ↔ Admin communication.

  • Who sent & who received
  • Message content
  • Type: Text, Leave request, Announcement
  • Read / Unread status
⚙️ Settings

School Configuration

General school settings.

  • School levels (Grade 1, 2…)
  • Subjects list
  • Time slots / timetable
  • Admin password
🕵️ History Log

Change History

Automatic record of all changes.

  • What was changed
  • Old value vs. new value
  • Date & time of change
